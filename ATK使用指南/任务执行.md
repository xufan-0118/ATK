# 单机执行

容器中需要先设定环境变量

**在测试性能时通过绑cpu核确保性能测试准确，即cpu执行时间、npu和gpu的host执行时间，不绑cpu核则该核不仅会用来进行性能测试，也会进行数据生成等，影响性能测试**

```shell
export ATK_BIND_CPU_TYPE=2   # 绑定cpu核的类型， 0为不绑定，1为物理机绑定，2为容器内绑定
```

* 执行下面命令进行精度比对：
  
  ```shell
  atk node --backend npu --devices 0 node --backend cpu task -c result/op_max/json/all_op_max.json --task accuracy -e 1
  ```
* 也可编写**node.yaml**文件：
  
  ```yaml
  nodes:
     - backend: npu
       task: ['accuracy']
       devices: [0]
     - backend: cpu
       task: ['accuracy']
  ```
  
  执行以下命令进行精度比对：
  
  ```shell
  atk task -c result/op_max/json/all_op_max.json -n node.yaml -e 1
  ```

# 多机在线执行

## 在远端（gpu侧）起服务

容器中需要先设定环境变量

```shell
export ATK_BIND_CPU_TYPE=2   # 绑定cpu核的类型， 0为不绑定，1为物理机绑定，2为容器内绑定
```

一定要确保无代理：

```shell
unset http_proxy https_proxy
```

启动服务

```shell
atk server --devices 0
```

> 1）此处默认启用的服务端口号是9090
> 
> 2）可同时启动多个server，通过命令`atk server --devices device_id -p port`指定不同的端口号，需确保不同server的端口不冲突
> 
> 3）在容器环境下每个使用的端口均需映射到宿主机上，如默认情况下需将9090端口映射到宿主机，并在节点配置文件中填写该端口号，如`9090->9471`，则填写`9471`
> 
> 4）自定义api执行和自定义数据类型，通过`--plugin_path xxxxxx`参数传入自定义文件或者路径

## 任务执行（npu侧）

容器中需要先设定环境变量：

```shell
export ATK_BIND_CPU_TYPE=2  # 绑定cpu核的类型， 0为不绑定，1为物理机绑定，2为容器内绑定
```

一定要确保无代理：

```shell
unset http_proxy https_proxy
```

* 执行测试任务：90.90.90.101为gpu服务器的ip地址，9471为物理机上映射到容器9090端口的端口号
  
  ```shell
  atk node --backend npu --devices 0 node --backend gpu -h 90.90.90.101 -p 9471 --devices 0  task -c result/op_max/json/all_op_max.json --task accuracy,performance_device -e 1
  ```
* 也可编写**node.yaml**文件：
  
  ```yaml
  nodes:
      - backend: npu
        task: ['accuracy','performance_device']
        devices: [0]
      - backend: gpu
        host: 90.90.90.101
        port: 9471
        devices: [0]
        task: ['accuracy','performance_device']
  ```
  
  > host字段只能在gpu侧（也就是开启server的一侧）设置
  
  执行以下命令进行精度和性能比对：
  
  ```shell
  atk task -c result/op_max/json/all_op_max.json -n node.yaml -e 1
  ```

# 多机离线执行

## 精度比对

**1. 保存gpu/npu标杆算子的输出结果**

```bash
atk node --backend gpu --devices 0 --name gpu_base node --backend cpu task -c result/torch.max/json/all_torch.max.json --task accuracy  --save_data output -s 0 -e 1
```

输出结果默认保存在`./atk_output/all_torch.max_2025-07-25-17-05-18-980232/output/`

**2. 导入标杆算子的输出结果，执行精度比对**

```bash
atk node --backend npu --devices 0 node --backend gpu --devices 0 --name gpu_base --output_path [第一步中的输出结果保存路径] task -c result/torch.max/json/all_torch.max.json --task accuracy -s 0 -e 1
```

## 性能比对

**1. 保存gpu/npu标杆算子的性能数据**

```bash
atk node --backend gpu --devices 0 --name gpu_base node --backend cpu task -c result/torch.max/json/all_torch.max.json --task performance_device -s 0 -e 1
```

得到输出结果excel文件，默认保存到路径`./atk_output/all_torch.max_2025-07-25-17-05-18-980232/report/all_torch.max_reports_2025-07-25-17-05-26.xlsx`

**2. 导入标杆算子性能数据作为基线，进行算子性能测试**

```bash

# pyaclnn场景
atk node --backend pyaclnn --devices 0 node --backend cpu --is_compare False node --backend gpu --devices 0 --name gpu_base --bm_file [第一步中的excel文件路径] task -c result/torch.max/json/all_torch.max.json --task performance_device -s 0 -e 1

# npu场景
atk node --backend npu --devices 0 node --backend gpu --devices 0 --name gpu_base --bm_file [第一步中的excel文件路径] task -c result/torch.max/json/all_torch.max.json --task performance_device -s 0 -e 1
```

得到输出文件，假如保存到路径output_test/all_torch.max_reports_2025-02-19-14-17-42.xlsx

【新增】如果性能数据中有多列基线，用户可通过设置`node --name auto`自动匹配当前芯片环境下的性能数据，也可通过设置其他name匹配对应的性能基线数据

```bash
# pyaclnn场景 auto自动匹配
atk node --backend pyaclnn --devices 0 node --backend cpu --is_compare False node --backend pyaclnn --devices 0 --name auto --bm_file [性能基线数据路径] task -c result/torch.max/json/all_torch.max.json --task performance_device -s 0 -e 1

# pyaclnn场景 指定name场景
atk node --backend pyaclnn --devices 0 node --backend cpu --is_compare False node --backend pyaclnn --devices 0 --name 910B3 --bm_file [性能基线数据路径] task -c result/torch.max/json/all_torch.max.json --task performance_device -s 0 -e 1
```

## `atk task`参数说明

### 基本功能

| 参数                                | 说明                                                         | 备注                                                         |
| ----------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| -c, --case                          | 必选 ，用例生成步骤中生成的json文件                          | `atk task -c *.json`                                         |
| -n, --nodes                         | 可选， node.yaml配置文件，不设置时需要设置node节点           | `atk task -n node.yaml`                                      |
| -l, --log                           | 可选，设置日志级别，["debug", "info", "warn", "error", "fatal"] | `atk task -l debug`                                          |
| -p，--plugin_path                   | 可选，支持python文件和目录，传入自定义API、自定义数据类型、自定义精度标准的执行文件路径 | `atk node --backend npu --devices 0 node --backend cpu task -c result/op_max/json/all_op_max.json --task accuracy -p api.py` |
| -tk, --task                         | 可选，任务组，若nodes里未设置task，则会替换，多个任务以逗号隔开，如 accuracy,performance_e2e。<br/><br/>可选参数及介绍：<br/><br/>**accuracy**：精度任务<br/><br/>**accuracy_load**：与已有结果的标杆精度文件作精度对比任务（需配置标杆文件存放的路径 output_path）<br/><br/>**performance_e2e**：端到端的性能任务，采集调用 `__call__` 起止的耗时，用于性能比较任务<br/><br/>**performance_device**：通过 profiling 采集 api 运行的 device 性能任务，采集 device 耗时、最大分配内存和最大预留内存；通过 `--save_data profile` 参数可保存 profiling 数据<br/><br/>**performance_device_pta**：通过 torch_npu/torch 的 memory 包采集 api 最大分配内存和最大预留内存信息，无 device 耗时<br/><br/>**memory_device**：通过 profiling 工具采集算子运行时最大分配内存和最大预留内存信息<br/><br/>**performance_camodel**：采集 camodel 的仿真性能数据<br/><br/>**accuracy_dc**：开启确定性计算，每个用例运行 50 次，如果一致则通过 | accuracy：`atk node --backend npu --devices 0 node --backend cpu task --task accuracy -c result/op_max/json/all_op_max.json`<br/>accuracy_load：`atk node --backend npu --devices 0 node --backend gpu --devices 0 --task accuracy_load --output_path /home/output -h 90.90.111.141 -p 9471 task --task accuracy -wl [4] -c result/op_max/json/all_op_max.json`<br/>performance_e2e：`atk node --backend npu --devices 0 node --backend cpu task --task performance_e2e -c result/op_max/json/all_op_max.json`<br/>performance_device：`atk node --backend npu --devices 0 node --backend cpu task --task performance_device -c result/op_max/json/all_op_max.json --save_data profile`<br/>performance_device_pta：`atk node --backend npu --devices 0 node --backend cpu task --task performance_device_pta -c result/op_max/json/all_op_max.json`<br/> |
| -mt,--max_task                      | 可选，设置算子测试最大并发任务数，默认100                    | `atk node --backend npu --devices 0 node --backend cpu task -c result/op_max/json/all_op_max.json --task accuracy -mt 1` |
| -cp, --cpp_func_signature_type_path | 可选，**pyaclnn测试使用**，是否进行pyaclnn参数校验，开启该参数会自动查找校验所需的头文件信息，优先级高于自定义执行文件内容 | `atk node --backend pyaclnn --devices 0 --name aclnn_base node --backend cpu task -c all_aclnnLogit.json --task accuracy -e 1 -cp` |
| -tup, --triton_ut_path              | 可选，**triton测试使用**，表示UT测试用例的路径，例如triton的UT测试文件目录为 ./generalization_cases，则需要传入-tup ./generalization_cases |                                                              |
| -xw, --xrun_whl                     | 可选，**xrun迁移ATK算子测试使用**，表示xrun需要的依赖包路径，支持py文件和目录 |                                                              |
| -ap, --atb_api_path                 | 可选，**atb测试使用**，表示atb的data_generation所在目录      |                                                              |
| -ecp, --experimental_config_path    | 可选，**自定义性能采集配置**，编写experimental_config.json文件（json格式-字典）,可定义参数（key）:aic_metrics,profiler_level,l2_cache,data_simplification |                                                              |
| --performance_data                  | 可选，**自定义性能采集次数参数**，必须按照顺序输入数字，依次是warmup_times(预热次数)，测试性能采集次数(per_times)，取后n次性能数据平均值(perf_ntimes),参数间用,隔开，可以不输入值采用默认值 | `atk node --backend pyaclnn --devices 0 --name aclnn_base node --backend cpu task -c all_aclnnLogit.json --task performance_device -e 1 --performance_data 400,50,36`  <br/> `atk node --backend pyaclnn --devices 0 --name aclnn_base node --backend cpu task -c all_aclnnLogit.json --task performance_device -e 1 --performance_data  ,50,` |
| **-sp,--single_process**            | 可选，设置单进程算子测试模式，目前已支持torch、pyaclnn上的精度比较、确定性计算、性能测试、内存检测任务，只需在命令末尾添加-sp，一键开启单进程 | `atk node --backend npu --devices 0 node --backend cpu task -c result/op_max/json/all_op_max.json --task accuracy -sp` |

### 数据保存和导入

| 参数                | 说明                         | 备注                                                                                                                                                                                                                                                                                                                   |
| --------- | -------------------------- | ------------------------- |
| --save_data                     | 可选 ["input", "output", "profile", "input_final"] ，表示是否保存input、output、profile、input_final（保存init_by_input_data处理后数据）数据, 默认均不保存，可同时指定多个保存类型。<br/>如--save_data input --save_data output，其中input和output会保存在`${output_path}`路径下，<br/>profile(需跑performance_device任务)会保存在`${output_path}/profiling_result/{api_name}`路径下<br/>当任务为accuracy_load时，可以将保存的数据传输到主节点上（路径不变），<br/>比如300IDuo和A2比较，可以将A2的输出数据传输到300IDuo上，并使用accuracy_load比较。<br/>另外新增input的txt保存形式可选，命令格式如input:bin、input:txt等 ，不指名则默认为bin文件                               | `atk task --save_data input --save_data input:txt`  |

# 自定义API执行方式

该部分主要定义了每个api在各个后端平台上是**如何通过生成的数据，执行生成输出结果**，同时返回值也会作为精度和性能对比测试的核心部分；

当前采用的策略是注册器+抽象类方法，可以通过新增文件继承基类来实现自定义API执行方式。

| method             | atk/tasks/api_execute/method_api.py                | method执行方式表示需先根据api_name实例化对象，再根据输入进行调用，如torch.nn.Conv2d等api。 |

### 抽象方法二：`__call__`

`__call__`方法是api的核心执行部分，该接口会在后端处调用，接收输入数据，执行后返回输出结果进行精度计算；
整个`__call__`方法的计算过程会记录耗时作为**e2e性能比对**的数据，不同的backend（cpu\npu\gpu）执行方式不一样时，需要针对性进行编写；
此外，`init_by_input_data`会先于`__call__`方法运行。

```python
class SampleApi(BaseApi):
    def __call__(self, input_data: InputDataset, with_output: bool = False):
        """
        :param input_data: 输入数据，具体属性值可参考InputDataset类变量：
                        args: yaml配置里inputs中未设置name的输入数据列表
                        kwargs: yaml配置里inputs中设置了name的输入数据列表
                        method_args: yaml配置里method_inputs中未设置name的输入数据列表
                        method_kwargs: yaml配置里method_inputs中设置l name的输入数据列表
                        tensor_args: yaml配置里的tensor_input输入数据
        :param with_output: 表示是否返回API执行后的返回值，测试性能时为False，测试精度时为True
        :return: None 或者 API的返回值
        """
        if not with_output:
            """
            此处实现不需要返回值时的自定义API调用方式
            """
            eval(self.api_name)(*input_data.args, **input_data.kwargs)
            return None

        if self.device == "cpu":
            """
            此处实现 cpu 的 API调用方式
            """
            output = eval(self.api_name)(*input_data.args, **input_data.kwargs)
        elif self.device == "npu":
            """
            此处实现 npu 的 API调用方式
            """
            output = eval(self.api_name)(*input_data.args, **input_data.kwargs)
        elif self.device == "gpu":
            """
            此处实现 gpu 的 API调用方式
            """
            output = eval(self.api_name)(*input_data.args, **input_data.kwargs)

        return output
```

### 抽象方法三：`cal_cube_computation`

cube计算量的计算方式参考：[关键算子MFU/TFLOPS计算公式](https://wiki.huawei.com/domains/68949/wiki/106372/WIKI202509028111944)
该方法的入参`task_duration, aicore_time, aic_mac_ratio`已经从profiling文件中采集完成，用户可以根据三个入参的数值计算得到`TFlops_e2e,TFlops_cube,MFU`，并按序返回这三个数值。

```python
class SampleApi(BaseApi): 
    def cal_cube_computation(self, task_duration, aicore_time, aic_mac_ratio):
        '''
        cube 计算量
        TFlops_e2e  = cube计算量(TFLOP) / 实际耗时(task_duration(s))
        TFlops_cube = cube计算量(TFLOP) / (aicoretime * aic_mac_ratio(s))
        MFU         = TFlops / 理论算力
        '''
        logging.info("cal_cube_computation() is not custom defined, "
                 "TFlops_e2e,TFlops_cube,MFU return 0,0,0 as default.")
        return 0, 0, 0
```

## 自定义API实现样例


### sample 样例

```python
import torch
from atk.configs.dataset_config import InputDataset
from atk.tasks.api_execute import register
from atk.tasks.api_execute.base_api import BaseApi
from atk.configs.results_config import TaskResult


@register("sample")
class SampleApi(BaseApi):
    def __init__(self, task_result: TaskResult):
        """
        BaseApi类的定义：
        self.task_result = task_result
        self.api_name = task_result.case_config.name
        self.output = task_result.case_config.outputs
        self.device = task_result.backend
        self.name = task_result.name
        self.device_id = task_result.device_id
        self.change_flag = False
        """
        super().__init__(task_result)
        # 可进行算子的初始化、变量初始化等操作
        # 工具会先执行init_by_input_data，再执行__call__，
        # 对于数据预处理等不需要计入到性能耗时的情况，
        # 可通过如下的方式用self.my_val保存，并在__call__里进行使用
        self.my_val = None

    def init_by_input_data(self, input_data: InputDataset):
        """
        :param input_data:
        :return:
        to modify input data
        """
        # 可进行输入数据的处理等操作
        # 如下实例，自行进行输入数据的构建
        if self.device == "cpu":
            import numpy as np
            self.my_val = np.random.rand(50, 5)
        
    def __call__(self, input_data: InputDataset, with_output: bool = False):
        # 自定义算子执行逻辑
        # 若json中输入name字段为空，则输入数据会存在input_data.args里
        # 若json中输入name字段非空，则输入数据会存在input_data.kwargs里，可以通过input_data.kwargs[name]字段调用

        # 可按需开启确定性计算
        torch.use_deterministic(True)

        # 工具会自动根据device类型将输入数据放到指定设备上
        # npu执行逻辑
        if self.device == "npu":
            import torch_npu
            # 根据设备id实现不同的执行逻辑
            if self.device_id == 0:
                output = torch_npu.arange(*input_data.args, **input_data.kwargs)
            elif self.device_id == 1:
                output = torch.arange(*input_data.args, **input_data.kwargs)
            else:
                # eval(self.api_name)即用json中name字段的值调用算子
                output = eval(self.api_name)(*input_data.args, **input_data.kwargs)
        # cpu执行逻辑
        elif self.device == "cpu":
            output = torch.arange(self.my_val)
        # gpu执行逻辑
        elif self.device == "gpu":
            # 获取json的指定name的输入
            input_x = input_data.kwargs["x"]
            # 如果没有指定名称，则根据index获取输入数据
            input_0 = input_data.args[0]
            output = torch.arange(input_x, input_0)
        return output 

```

```python
class BaseRunModeFunc(ABC):

    @abstractmethod
    def begin_func(self, backend):
    """
    from tasks.backends.backend import BaseBackend
    backend: BaseBackend
    执行前置修改
    """
    pass

    @abstractmethod
    def end_func(self, backend):
    """
    from tasks.backends.backend import BaseBackend
    backend: BaseBackend
    执行后恢复
    """
    pass
```

定义类变量funcs，并实现funcs实例中的抽象类接口。
