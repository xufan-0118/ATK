# 模型采集算子测试指南

[toc]

---

**模型的单算子测试，与模型中算子实际运行的shape、数据类型、取值范围等不能保持完全一致，因此工具支持将模型动态运行的dump数据，转换成ATK工具能够执行的json测试用例，从而实现模型算子的测试。**

具体过程分为以下几个步骤：

1、模型dump数据转换成atk工具需要测试用例

2、使用ATK工具执行gpu侧算子测试并保存输出数据作为golden数据

3、使用atk工具在npu侧执行算子测试，通过加载gpu侧保存的golden数据，实现npu+cpu+gpu双标杆对比测试


# 测试用例生成

本节需要通过atk工具，生成atk测试需要用到测试用例，即一个json文件。输入为模型数据dump下的json文件，输出为atk测试需要的包含用例信息的json文件。

## dump模型数据

通过MindStudio的msprobe工具，在模型的训练脚本内添加dump接口并启动模型训练，来采集模型的精度数据（只需要采集一个step即可）。

msprobe工具安装和使用参考链接：[PyTorch 场景的精度数据采集](https://gitcode.com/Ascend/mstt/blob/master/debug/accuracy_tools/msprobe/docs/zh/dump/pytorch_data_dump_instruct.md)

**（1）msprobe工具采集精度数据示例代码**

```python
from msprobe.pytorch import PrecisionDebugger

# 请勿将PrecisionDebugger的初始化流程插入到循环代码中
debugger = PrecisionDebugger(dump_path="./dump_path")

# 模型、损失函数的定义及初始化等操作
# ...

# 数据集迭代的位置一般为模型训练开始的位置
for step in steps:
    debugger.start()  # 开启数据dump

    # 如下是模型每个step执行的逻辑
    # ...

    debugger.stop()  # 关闭数据dump
    debugger.step()  # 结束一个step的dump
```

**（2）参考示例修改并执行训练脚本，获取算子信息文件**

算子信息文件`dump.json`：保存模型中前反向算子、算子的统计量信息或溢出算子信息。

文件包含dump数据的API名称（命名格式为：`{api_type}_{api_name}_{API调用次数}_{前向反向}_{input/output}.{参数序号}`）、dtype、
shape、各数据的max、min、mean、L2norm统计信息以及当配置summary_mode="md5"时的md5数据。

## 生成用例

将dump的模型算子信息文件`dump.json`作为输入，通过ATK工具执行测试用例转换，生成测试用例文件`torch_case.json`和`torch_npu_case.json`。

执行命令如下：

* **测试用例生成示例**

```shell
atk dump -f dump.json --torch_output_json torch_case.json --torch_npu_output_json torch_npu_case.json --split_path dump_test -as new_benchmark
```

**atk dump命令的参数说明如下：**

| 参数                        | 说明                                                | 备注                                    |
|---------------------------|---------------------------------------------------|---------------------------------------|
| `-f`,` --case_file`       | 输入dump的json文件路径                                   | 必选                                    |
| `--torch_output_json`     | 输出的torch算子用例                                      | 可选，默认为`torch_case_{时间戳}.json`         |
| `--torch_npu_output_json` | 输出torch_npu算子用例<br>（仅在NPU上支持的算子）                  | 可选，默认为`torch_npu_case_{时间戳}.json`     |
| `--split_path`            | 对输出的json进行切分，设置切分后文件的保存路径                         | 可选，默认为`split`                         |
| `-as`, `--acc_standard`   | 使用的精度对比方法                                         | 可选，默认为`single_bm`                     |
| `-ps`,`--perf_standard`   | 使用的性能对比方法                                         | 可选，默认为`not_key`                       |
| `--api_type`              | 传入json文件，对函数的api_type进行转换                         | 可选                                    |
| `-bl`,`--black_list`      | 用例清洗的关键字列表，支持*号模糊匹配，<br>多个关键字用逗号隔开。示例：`add,__*__` | 可选                                    |
| `-gt`,`--gen_type`        | 生成用例的方式                                           | 可选，forward\|backward\|all             |
| `-l`,`--log`              | 日志级别                                              | 可选，debug\|info\|warning\|error\|fatal |

# 自定义算子执行

在测试自研torch_npu算子、标杆实现需要自定义（如融合算子）等场景中，想要自定义标杆的执行逻辑，则需修改`api_type`字段，并编写算子执行逻辑的自定义文件。

完整的指导请参考：[自定义api执行](../任务执行.md#自定义api执行方式)

**自定义算子执行逻辑的示例**如下：

```python
import torch

from atk.configs.dataset_config import InputDataset
from atk.tasks.api_execute import register
from atk.tasks.api_execute.base_api import BaseApi
from atk.configs.results_config import TaskResult


@register("sample")
class SampleApi(BaseApi):
    def __init__(self, task_result: TaskResult):
        """
        BaseApi类的定义：
        self.task_result = task_result
        self.api_name = task_result.case_config.name
        self.output = task_result.case_config.outputs
        self.device = task_result.backend
        self.name = task_result.name
        self.device_id = task_result.device_id
        self.change_flag = False
        """
        super().__init__(task_result)
        # 可进行算子的初始化、变量初始化等操作
        # 工具会先执行init_by_input_data，再执行__call__，
        # 对于数据预处理等不需要计入到性能耗时的情况，
        # 可通过如下的方式用self.my_val保存，并在__call__里进行使用
        self.my_val = None

    def __call__(self, input_data: InputDataset, with_output: bool = False):
        # 自定义算子执行逻辑
        # 若json中输入name字段为空，则输入数据会存在input_data.args里
        # 若json中输入name字段非空，则输入数据会存在input_data.kwargs里，可以通过input_data.kwargs[name]字段调用

        # 可按需开启确定性计算
        torch.use_deterministic_algorithms(True)

        # 工具会自动根据device类型将输入数据放到指定设备上
        # npu执行逻辑
        if self.device == "npu":
            import torch_npu
            # 根据设备id实现不同的执行逻辑
            if self.device_id == 0:
                output = torch_npu.arange(*input_data.args, **input_data.kwargs)
            elif self.device_id == 1:
                output = torch.arange(*input_data.args, **input_data.kwargs)
            else:
                # eval(self.api_name)即用json中name字段的值调用算子
                output = eval(self.api_name)(*input_data.args, **input_data.kwargs)
        # cpu执行逻辑
        elif self.device == "cpu":
            output = torch.arange(self.my_val)
        # gpu执行逻辑
        elif self.device == "gpu":
            # 获取json的指定name的输入
            input_x = input_data.kwargs["x"]
            # 如果没有指定名称，则根据index获取输入数据
            input_0 = input_data.args[0]
            output = torch.arange(input_x, input_0)
        return output

    def init_by_input_data(self, input_data: InputDataset):
        """
        :param input_data:
        :return:
        to modify input data
        """
        # 可进行输入数据的处理等操作
        # 如下实例，自行进行输入数据的构建
        if self.device == "cpu":
            import numpy as np
            self.my_val = np.random.rand(50, 5)
```

# 测试执行

通过dump文件转换的搭配的`torch_case.json`文件，直接输入到atk工具中即可实现模型dump的算子测试。

## 生成gpu侧标杆

如果需要跟gpu进行比较，则需要先再gpu服务器上生成标杆的输出，执行命令示例如下：

- **gpu标杆生成**

```shell
atk node --backend gpu --devices 0 task -c torch_case.json --task accuracy --save_data output
```

执行成功后gpu测试结果保存的路径会打屏，即`save output path`所显示路径，如`/home/atk_output/torch_case_*/output`。

`output`文件夹即为保存下来gpu测试结果，将该文件夹下载到npu服务器上，进行npu算子测试。

## 执行算子测试

- **npu vs cpu 精度测试**, 不需要gpu侧标杆, 执行命令示例如下：

```shell
atk node --backend npu --devices 0 node --backend cpu task -c torch_case.json --task accuracy
```

- **npu vs gpu 精度测试**, 其中gpu数据需要从`output_path`目录导入, 执行命令示例如下：

```shell
atk node --backend npu --devices 0 node --backend gpu --devices 0 --task accuracy_load --output_path /path/to/output task -c torch_case.json --task accuracy
```

> 其中`/path/to/output`是从gpu侧上保存输出数据到npu上的文件路径
>
> 执行成功后，会打屏回显任务的执行结果，也会回显记录执行结果的文件路径，即`save result excel path`所显示路径

执行命令中的完整参数说明请参考：[参数说明](../任务执行.md#参数说明)

# profiling文件转换测试用例

除了msprobe工具采集的`dump.json`之外，工具也支持将profiling报表`op_summary_xxx.csv`文件导出为atk的测试用例，工具命令示例如下：

```shell
atk dump -pf op_summary_xxx.csv
```

**详细参数说明：**

| 参数                           | 说明                                                | 备注                                    |
|------------------------------|---------------------------------------------------|---------------------------------------|
| `-pf`,` --profile_case_file` | 输入profiling文件路径， 支持csv/xlsx格式                     | 必选                                    |
| `--torch_output_json`        | 输出的torch算子用例路径                                    | 可选，默认为`执行路径/torch_case_{时间戳}.json`    |
| `-cf`, `--custom_file`       | 自定义设置精度标准、性能标准、api_type等                          | 可选，默认为`single_bm`+`not_key`+`function`  |
| `-bl`,`--black_list`         | 用例清洗的关键字列表，支持*号模糊匹配，<br>多个关键字用逗号隔开。示例：`add，__*__` | 可选                                    |
| `-l`,`--log`                 | 日志级别                                              | 可选，debug\|info\|warning\|error\|fatal |

**`-cf`, `--custom_file`  参数自定义json文件格式如下：**

```json
{
  "aclnnMatmul": {
    "acc": "single_bm",
    "perf": "not_key",
    "api_type": "function",
    "aclnn_api_type": "aclnn_matmul"
  },
  "aclnnAdd": {
    "acc": "double_benchmark",
    "perf": "not_key",
    "api_type": "function",
    "aclnn_api_type": "aclnn_add"
  }
}
```
