**statistic Sheet**

| 列名|说明 |
| --- | --- |

| 列名|说明 |
| --- | --- |

# 精度标准说明

- 针对单标杆和双标杆的精度标准，如果输出中包含除了**bfloat16、float16、float32**之外的数据类型，则根据以下规则进行比较：
  **1. 输出是int8、uint8，精度标准用量化计算；**
  **2. 输出是bool、int16、int32、int64等，精度标准用二进制一致；**
- 如果需要对精度标准的阈值进行修改，请参考：[设置自定义参数](#设置自定义参数)

| 精度场景            | 使用方式                               | 说明                                                         |
| ------------------- | -------------------------------------- | ------------------------------------------------------------ |
| 二进制比对          | acc: md5                               | 直接比较两个输出结果的 md5 值，若相等则精度通过，否则精度不达标 |
| 单标杆比对-高精度   | acc: single_bm                         | 通过 CPU 上更高精度的标杆数据作为标准，以相对误差和绝对误差分别对应大值域/小值域进行精度对比，同时判断误差均衡性。<br/>默认高精度模式，可自定义设置高精度/高性能/vector类融合算子标准 |
| 单标杆比对-高性能   | acc: single_bm: type: high_performance | 计算方式同高精度，但标准阈值更宽松                           |
| 双标杆比对          | acc: cv_fused_double_benchmark         | 通过 CPU 上更高精度标杆作为真值，分别计算最大相对误差、平均相对误差、均方根误差和误差均衡性，用于对比 NPU 和 GPU 的结果 |

# 自定义精度比较脚本适配

## 实现自定义场景
工具支持新增自定义精度标准，通过`-p`参数传入即可，`equal_compare.py`样例如下
```python
import torch

from atk.configs.results_config import AccuracyConfig
from atk.tasks.post_process.base_compare import BaseAccuracyCompare
from atk.tasks.post_process import ACCURACY_REGISTRY
from atk.common.utils import safe_equal


@ACCURACY_REGISTRY.register("equal")
class Compare(BaseAccuracyCompare):
    
    # 自定义实现compute_accuracy_result接口，然后将结果填入到AccuracyConfig中，并返回该实例
    # 参数说明 local_output：npu 输出 tensor
    # 参数说明 remote_output：标杆cpu/gpu 输出 tensor
    # 参数说明 data_file：保存 tensor 的原始文件路径
    def compute_accuracy_result(self, local_output, remote_output, data_file):
        if torch.isnan(local_output).all() and torch.isnan(remote_output).all():
            ret = AccuracyConfig(
                filename=data_file,
                result=True,
                error_info="torch.equal pass"
            )
            return ret
        if safe_equal(local_output, remote_output):
            ret = AccuracyConfig(
                filename=data_file,
                result=True,
                error_info="torch.equal pass"
            )
        else:
            ret = AccuracyConfig(
                filename=data_file,
                result=False,
                error_info="torch.equal failed"
            )
        return ret
```

1. 在自定义文件中继承基类BaseAccuracyCompare，并注册到ACCURACY_REGISTRY中，其中注册的名称用于在api的配置yaml文件中通过standard->acc字段指定。
2. 实现抽象类接口compute_accuracy_result，该接口接收三个参数：local_output, remote_output, data_file，其中local_output, remote_output对应本地和被比对的输出数据，是已经加载且拉平成一维的tensor数据，data_file则是该输出对应的文件名。
3. 该接口需要返回的是一个AccuracyConfig()类型数据，其属性result表示该次比对的结果是否通过，error_info用于说明精度不达标的具体信息。
4. 若api有多个输出，基类中会循环遍历每个输出文件依次调用compute_accuracy_result接口，逻辑保持一致。

## 设置自定义参数

若要在精度比较执行时传入自定义的参数，可以在用例设计时在api的yaml配置文件中设置自定义参数，然后在精度比较时可以获取该参数，下面以新精度标准的可配置项为例：


### 单标杆设置自定义阈值（single_bm）

- `single_bm`支持fp16、bf16、fp32三种dtype的误差、eb阈值自定义配置

```yaml
standard:
  acc:
    single_bm:
      # type 选择高精度/高性能/Vector融合算子标准类型，默认为高精度 
      # 高精度:high_precision;高性能:high_performance;Vector融合算子:vector_fused
      type: high_performance   
      fp16_error: 2**-8             # float16、自定义误差阈值
      fp16_eb: xxx                    # float16、自定义EB阈值
      fp32_error: xxx
      fp32_eb: 2**-13
```

### 双标杆设置自定义阈值（cv_fused_double_benchmark）

- `cv_fused_double_benchmark`支持fp16、bf16、fp32三种dtype的最大相对误差、平均相对误差、均方根误差、eb阈值自定义配置
- 各看护等级阈值参考：[精度标准2.0更新报告](https://wiki.huawei.com/domains/62672/wiki/219231/WIKI202507117472298?title=_a246e62a)

| 精度等级        |最大相对误差 <br>max_re_ratio| 平均相对误差 <br> avg_re_ratio | 均方根误差 <br> root_mean_squared_ratio |
|-------------| --- | --- | --- |
| L0级精度看护（默认） | 10 | 2 | 2 |
| L1级精度看护     | 5 | 1.5 | 1.5 |
| L2级精度看护     | 2 | 1.2 | 1.2 |

```yaml
## 高性能双标杆 & CV融合类算子(cv_fused_double_benchmark)
standard:
  acc:
    cv_fused_double_benchmark:
	  max_re_ratio: 10
      avg_re_ratio: 2
      root_mean_squared_ratio: 2
```

### 性能标准

```yaml
standard:
  perf: 
    not_key:
      [0.95, 0.95]
```

配置yaml文件中的standard->perf参数控制， 默认为not_key。每个性能标准有两个数值，每个数值的计算意义如下：

* 第一位：第一个数据表示该算子的所有用例的平均的性能比值需大于这个阈值，如10个用例，每个用例都有一个性能比值，则10个用例的平均的性能比值需要大于该阈值。
* 第二位：第二个数据表示该api的每个用例的性能比值均需要满足大于这个阈值，如10个用例，每个用例都有一个性能比值，则10个用例性能比值均需要满足大于这个阈值。


# 自定义精度标准实践案例

## 精度比对时，同时需要用到高精度、低精度的标杆输出值、入参

**场景**：
NPU和CPU比对，且CPU的高精度和低精度输出都需要用于精度计算，且计算的时候还要用到一些int/float/string类型的入参。
**思路**：
atk默认情况下会将输出的每个tensor分别落盘 比对的时候分别成对的进行tensor的比对 这会导致：

1. 丢失非tensor数据
2. 丢失入参
3. 要求出参的数目一致且出参需对应 不然比对会出问题
   因此我们可以直接绕过atk原有的数据落盘和精度比对逻辑，手动分两步实现这个过程

**实现方式**：

1. 在自定义执行文件中，先获取保存所需的路径，然后手动保存**所有用于精度比对**的数据到list中。

```python
import torch

from atk.configs.dataset_config import InputDataset
from atk.tasks.api_execute import register
from atk.tasks.api_execute.base_api import BaseApi
from atk.common.connection import RemoteManager


@register("function")
class FunctionApi(BaseApi):
    def __call__(self, input_data: InputDataset, with_output: bool = False):
        # 获取不同node的输出目录，可以直接复制进自己的代码中
        post_node = self.task_result.nodes.get_main_node()
        remote_nodes = []
        for node in self.task_result.nodes.nodes:
            if node != post_node:
                remote_nodes.append(node)
        remote_manager = RemoteManager(self.task_result.case_config, post_node,remote_nodes[0] )
        local_dir, remote_dir = remote_manager.get_local_and_remote_dir()
        # 算子执行逻辑，以下仅是示例
        result1, result2  = eval(self.api_name)(*input_data.args, **input_data.kwargs)
        # 保存用于精度比对所需的各种数据，以下仅是示例，务必把所有的tensor都.cpu()
        if self.device == 'npu': 
            torch.save([result1.cpu(), result2.cpu(),  input_data.kwargs['xxx'] ] , f"{local_dir}/result.bin")
        elif self.device == 'cpu':
            torch.save([result1.cpu(), result2.cpu(),  input_data.kwargs['xxx'] ] , f"{remote_dir}/result.bin")
        return result1
```

2. 在自定义精度比对文件中处理比对逻辑
